# Интернет магазин

## Описание проекта
Интернет-магазин на Django, позволяющий пользователям просматривать каталог товаров, создавать и управлять статьями в блоге, а также регистрироваться и редактировать свой профиль. Проект решает задачу создания полнофункционального веб-приложения с пользовательскими ролями, модерацией контента и управлением товарами.

## Полное описание функционала

### Каталог товаров
- Просмотр всех опубликованных товаров на главной странице
- Просмотр товаров по категориям
- Создание, редактирование и удаление категорий (для пользователей с соответствующими правами)
- Создание, редактирование и удаление товаров (владельцем или модератором)
- Публикация и снятие с публикации товаров (владельцем или пользователем с правом модерации)
- Загрузка изображений для товаров

### Блог
- Просмотр активных статей на главной странице блога
- Просмотр архивных (неопубликованных) статей (для авторизованных пользователей)
- Создание, редактирование и удаление статей
- Просмотр детальной информации о статье с подсчетом просмотров
- Загрузка изображений для статей

### Пользователи
- Регистрация новых пользователей
- Авторизация и выход из системы
- Просмотр, редактирование и удаление профиля
- Загрузка аватара профиля
- Отправка сообщений через форму обратной связи (вывод в консоль сервера)

### Администрирование
- Полный доступ к управлению товарами для суперпользователя
- Права модерации для определенной группы пользователей

## Архитектурные решения и особенности проекта

### Модели Django

**Модель Category (Категория)**
- Связана с Product через ForeignKey
- Поля: name (уникальное имя), description, updated_at

**Модель Product (Товар)**
- Связана с Category через ForeignKey (одна категория - много товаров)
- Связана с CustomUser (владелец) через ForeignKey
- Поля: name (уникальное имя), description, image, category, price, created_at, updated_at, published, owner
- Имеет пользовательское разрешение "can_unpublish_product" для модерации

**Модель BlogEntry (Статья блога)**
- Связана с CustomUser (автор) через ForeignKey
- Поля: title (уникальное имя), entry, image, created_at, is_active, view_counter
- При просмотре статьи автоматически увеличивается счетчик просмотров

**Модель CustomUser (Пользователь)**
- Наследуется от AbstractUser
- Дополнительные поля: email (уникальный, основной идентификатор), phone_number, avatar, country
- Использует email в качестве USERNAME_FIELD

### Особенности реализации
- Использование кэширования для представлений (cache_page декоратор)
- Реализация миксинов для обработки изображений (ImageHandlingMixin, AvatarHandlingMixin)
- Использование встроенной системы разрешений Django для контроля доступа
- Группы пользователей ("Модератор продуктов") для управления правами
- Отправка приветственного email при регистрации (с использованием send_mail)
- Пагинация для списков товаров, категорий и статей
- Использование кастомных шаблонов и наследования шаблонов (base.html)

## Инструкции по запуску

### Клонирование репозитория
```bash
git clone https://github.com/alekseyzvyagintsev/middle_web_app.git
```

### Настройка окружения
```bash
cd middle_web_app
poetry install
```
или
```bash
pip install -r requirements.txt
```

### Настройка переменных окружения
1. Создайте файл `.env` в корне проекта на основе примера:
```bash
cp .env.example .env
```
2. Заполните необходимые переменные в `.env` файле:
   - SECRET_KEY: секретный ключ Django
   - DEBUG: режим отладки (True/False)
   - ALLOWED_HOSTS: разрешенные хосты (например, '*')
   - DB_HOST: HOST для подключения к базе данных (по умолчанию 'localhost')
   - EMAIL_HOST_USER: email для отправки писем (обратитесь к описанию настройки почтового сервера)

### Применение миграций
```bash
python manage.py migrate
```

### Создание суперпользователя (опционально)
```bash
python manage.py createsuperuser
```

### Заполнение базы данных тестовыми данными (опционально)
```bash
(создание групп и пользователей admin, moderator, user. У всех пользователей password = zxcm3490)
python manage.py cau

(заполнение каталога)
python manage.py fill_blog

(добавление товаров)
python manage.py add_products

или смотри Загрузка фикстур
```

### Загрузка фикстур (опционально)
```bash
(загрузка групп)
python manage.py loaddata fixture_group.json

(загрузка пользователей)
python manage.py loaddata fixture_users.json

(загрузка категорий и товаров)
python manage.py loaddata fixture_catalog.json

(загрузка статей)
python manage.py loaddata fixture_blog.json
```

### Запуск сервера
```bash
python manage.py runserver
```

### Открытие в браузере
Перейдите по адресу:
```
http://127.0.0.1:8000/
```
